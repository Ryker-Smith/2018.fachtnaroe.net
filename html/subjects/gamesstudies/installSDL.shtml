Install SDL

To install perl SDL on Linux is a little time consuming, but worth it.

You'll need some sample programs (if you can run them it means perl/SDL is working). You'll have to have superuser access, and a mug of coffee.

Start a console session and try to run one of your sample perl programs. You'll probably get at error message such as:

Can't locate SDL.pm in @INC (@INC contains: lots of stuff shown here) at programName.pl line 100 (or some other number)

That's the first of a series of errors you'll see, from which you'll learn what needs to be installed. After each command below, you'll have to wait a while as software is downloaded and installed.

# first make sure kit is up-to-date
yum upgrade 
# now make sure perl is installed (should be already, right?)
yum install perl
# now add the SDL stuff (copy and paste this)
yum install SDL SDL-devel SDL-* SDL_* perl-Alien-SDL perl-SDL

Are you done installing stuff? Probably not. Try to run your test program again and see does it work.

If you get errors similar to that given at the start of this page, you can try asking Mr Google. But you can also try:

# to see what's installed for SDL
yum list installed | grep SDL

If there's something missing, yum it in as well. To see what's available try:

# to see what's available that might be missing
yum list available | grep SDL

If there are still errors, try:

# to see what files are in a package:
repoquery -ql  package_name
# to see what provides a missing file:
yum provides */missing_filename

At all times, read the error messages, and think your way through the problem. Copying the error messages to Mr G. helps when you get stuck.
