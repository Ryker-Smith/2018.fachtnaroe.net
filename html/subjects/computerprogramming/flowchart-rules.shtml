Flow Chart Rules

Rules

  1. Every box should have arrows in and out. Exceptions: Start box and End box.
  2. No box should have more than one arrow in.
  3. No box should have more than one arrow out. Exception: The decision box.
  4. The decision box has two arrows out.
  5. Each box should correspond with one action.
  6. A parallelogram requesting data should have a corresponding parallelogram accepting the data.
  7. Request boxes correspond with printf statements.
  8. Accept boxes correspond with scanf statements.
  9. All boxes should be in sequence joined by lines with arrows to show the sequence.
  10. Boxes handling data should use the name of the variable in which the data is stored.
  11. Use parallelograms for I/O statements.
  12. Use Rounded rectangles for processing.
  13. Use circles for Start and End boxes
  14. Use diamonds for decision, including loop completed decisions (while).

Purpose

The use of flowcharts allow for an unambiguous expression of the sequence of instructions of the program. This is also know as the logic of the program. Whilst this method is not fool-proof it is an invaluable tool for getting your mind to approach a problem in a straight line.
