    The Java Bridge and your first Android App (Android Studio Version)

    The App Inventor Java Bridge is a Java library for building Android apps. It has a much easier learning curve than the Android SDK, which is the library commonly used in Android development. The Java Bridge has an added advantage for those who have programmed in App Inventor, as the terminology is the same: there is a Java class pertaining to each App Inventor component, and the methods of each class correspond to App Inventor component properties or functions. 

    Each Java class has a data member for every property in the component which can be set/get with the name of the property. For instance, to create a Button, you'll code:

       Button button1;
       button1 = new Button(this);   // this refers to the enclosing class, the Screen (button is a 'child' of it)
       button1.Text("Click me");

    The code above is equivalent to dragging a Button component into the designer in App Inventor, then setting its Text property to "Click me".

    You won't need to create classes for the app we'll build, you'll use the ones provided in the library.   

    In this lesson, you’ll learn to build Android apps using the Android Studio and the App Inventor Java Bridge library.

    Getting Started

    You need an interactive development environment (IDE) to build Android apps. In this tutorial, you'll use Android Studio. Before installing Android Studio, you'll need Java 8  so check your computer to see if the Java 8 SDK is installed (you can check on a terminal with "java -version". If you don't have Java 8 (Java 1.8.x), install it. (Links to an external site.)Links to an external site.

    Next, follow the instrutions at install Android Studio (Links to an external site.)Links to an external site. 

    Your First App: Click Red

    For your first experience with the Java Bridge library, you’ll create perhaps the simplest app: when the user clicks on a button, it changes color to red (Wahoooooo!). The sample demonstrates the structure you’ll need in your code to build Android apps with the Java Bridge library.

    1. Create a new Android Application Project. Choose File | New Project and name your application "RedClick". Click Next, then  leave the default minimum API version of 8 (Anroid version 2.2 Froyo). Ignore the other items and click Next.  On the next screen, choose a Blank Activity for the app and on the final screen just click Finish.

    2. Run the App you have so far. You should now have the skeleton app that Android Studio sets you up with, one that displays “hello world”. You can run the app by selecting the project and clicking the green arrow icon  in the top menu. If your Android device is plugged in to your computer with USB, you can run it on that. If not, you can run it in an emulator. To test on the emulator, you may need to setup an Android Virtual Device (AVD). Just choose any device and run the app.

    3. Import the Java Bridge Library. Next, you'll customize the app to add the Java Bridge library and code for the sample click a button and make it red. Download this jar file to your computer. You need to copy it into the libs directory of your Android Studio project. To do this, first choose the Project view for your project. You can do this by clicking on the "Android" above the left panel that shows your project items, and choosing "Project" Then click on the "app" folder and you'll see a folder named libs. Open a file manager, find the .jar file you just downloaded, and drag it into the libs folder. Then right-click (or control-click) on the .jar file and choose "Add as Library". Your app can now use the Java Bridge library and all the App Inventor components.

    4. Import Java sample code: Click Red. Download this Screen1.java file to your computer (you may have to copy the code in the file and paste it into a new edit file). Then open the folder src/main/java/com.example.yourname.redclick in your project, and drag Screen1.java into it. This Screen1.java code will serve as a replacement for MainActivity.java. It should be fine, but just you may need to change is the “package” specification at the top of the file. Copy the package specification from MainActivity.Java, e.g., “package com.example.clickred;” and paste it to replace the package specification in the sample file.

    5. Modify the Android Manifest File. Your  Android project has a manifest file which specifies various things about your app. Go back to the Android View in your left-panel, then choose the manifests directory and click on the AndroidManifest file to open it. You want to change it so that the Screen1 class you just imported is used as the initial Activity (screen) instead of MainActivity. In the file, you’ll see the following:
           <activity android:name=".MainActivity"

            Replace “.MainActivity” with “.Screen1” (leave the initial dot),  so it appears as:
           <activity android:name=".Screen1"

    6. Rerun the app. If all is well, the app should now appear with a single button that when clicked, turns red. You've completed your first Android app

    EXERCISES

    1. Modify the app so that:

    • the button starts out showing a '1'.
    • each time the button is clicked, the number on it is incremented (1 to 2, 2 to 3, etc.).

    You'll need to convert String values to numbers and then back to String. Here are example calls to Java conversion methods:

          int score = Integer.parseInt("323");
          String s = String.valueOf(73);
