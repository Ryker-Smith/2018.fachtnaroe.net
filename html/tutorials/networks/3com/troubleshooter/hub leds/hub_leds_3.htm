Alert LED is ON:

The Alert LED comes on when one of the ports has partitioned or when there is excessive network traffic.

One of the ports has Partitioned

If one of the ports has partitioned or become temporarily disabled the OfficeConnect device has either detected an illegal network loop, or faulty equipment is connected to one of the ports.

Network loops occur when there is more than one path through the network to a device. The OfficeConnect device temporarily partitions (disables) a port to make sure that that there is only one path. Partitioning the port is only a temporary solution. To resolve the problem you must reconfigure the network to make sure that there are no network loops.

Faulty equipment may include broken cables or NICs/PCMCIA cards.

Make sure that the equipment attached to that partitioned port is turned on and that the cable is securely connected at both ends of the connection. If this fails to solve the problem you may need to systematically replace equipment you suspect could be broken with equipment you know to be working. For example, replace the cable with one you know works. If that does not solve the problem, check the physical installation and configuration of the NIC or PCMCIA card. Common NIC problems include poor physical installation (the NIC may not have been pushed fully into the slot) and incorrect configuration of protocols.

You can check which ports are partitioned, by looking at the Port Status LEDs.

Excessive network traffic

If, for example, 80% of the available bandwidth is being used for several seconds at a time, excessive network traffic is responsible for the Alert LED being ON. If your OfficeConnect device has Utilization LEDs, use them to check the amount of bandwidth being used. You can:

  • Use Ethernet Switches to divide the network into smaller less congested sections.

  • Use Fast Ethernet to provide more bandwidth.
