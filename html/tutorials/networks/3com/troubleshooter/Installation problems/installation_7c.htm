Cannot See the Device On the Network When Looking in Network Neighborhood:

Computers on your network will only be able to communicate if they are using the same network protocol. For example, a PC that is using TCP/IP will not appear in the same Network Neighborhood as a PC using NetBEUI.

Note: This is a particular problem if you have a network that has both Windows 95 and Windows 98 computers. Windows 95 uses NetBEUI by default and Windows 98 uses TCP/IP by default. You may wish to configure your Windows 95 computers to use TCP/IP if they need to communicate with Windows 98 computers or with routers and modems that require IP. Setting up TCP/IP is described in Setting up IP in this OfficeConnect Network Assistant.

If protocol compatibility is not the problem:

  1. Make sure that the PC or laptop you want to access is switched on, and that the laptop is not in sleep mode (conserving energy).

  2. Make sure that the PC or laptop you are trying to access is set up for File Sharing, and have the same Workgroup name as other computers on your network:
    1. From the Start menu, choose Settings and then Control Panel.

    2. In the Control Panel window, double-click Network.

    3. In the Configuration tab, click File and Print Sharing.

    4. In the File and Print Sharing dialog box, select both of the checkboxes.

    5. Click OK.

    6. Select the Identification tab.

    7. Check that there is a unique name in the Computer name field.

    8. The name entered in the Workgroup field must be the same workgroup name entered on other PCs and laptops on the network.

    9. Click OK.

    10. Restart the PC or laptop.

  1. When a new device is added to the network, it may take the Microsoft Browse Master some time to recognise that the new device has been added. Instead of waiting for the your PC's network information to be updated, you can use the Find option to find a specific PC or all PCs on your network:
    1. Click on the Start Menu.

    2. Click on the Find option.

    3. Click on Computer option.

    4. In the Named field enter the name of the computer you want to find. The results of the search are shown in the Find listbox. If the computer is not shown, go to step 4.

  1. To check the communication path between the devices that are trying to communicate, carry out the following steps:
    1. Make sure that your own and the remote PC or laptop are still connected to the hub, and that the ends of the cable are firmly pushed into the sockets on the NIC/PCMCIA card and hub.
    2. Check that the Power LED on the OfficeConnect device is lit (refer to the user guide or tutorial if necessary).

    If the Power LED is not lit, make sure that:

      • A 3Com approved and working power cable is connected to the OfficeConnect device. Refer to your supplier for details on how to purchase the correct power cable.
      • The power cable is also connected to a working main power supply socket.
        Tip: You can check that a socket is working by plugging a device you know that works into the socket, for example, a lamp.
      • If the main power supply socket has a switch, make sure that it is switched to the ON position.
      • If this does not solve the problem, the hub may be faulty. Disconnect the device from the power supply, and refer to your supplier for more details on warranty and service.

    1. If the power LED is lit, check that the Port Status LED on the hub is lit for both your own computer and the remote PC connection. If either of the Port Status LEDs are OFF, flashing Green for the Dual Speed Hub, or Yellow, then you need to carry out the following steps:

      • Replace the cable between that PC/laptop and the hub with a cable you know works. If the LED then comes on, the original cable was broken.
        Tip: Make sure that you are using the correct cable type. A common mistake is to use telephone cables. Check Connectors and Cabling for more details.
      • If the PC or laptop is connected to the highest numbered port on the hub, the MDI/MDI-X switch may be incorrectly set. Toggle the MDI/MDI-X switch to see if the Port Status LED lights.
      • Make sure that the NIC or PCMCIA card and driver software are correctly installed. If necessary, de-install the NIC/PCMCIA card and drivers, and re-install them. Refer to Changing the Drivers.

    2. Check that there are no other broken cables between the devices that are trying to see each other on the network.

    3. If there is more than one hub, the MDIX switch may be incorrect.

      • If the MDIX ports on the hubs are used to connect the hubs, toggle the MDI/MDI-X switch on one of the Hubs.
      • If a normal port is used to connect one hub to the MDIX port on another hub, toggle the MDIX switch on the hub which uses the MDIX port.

      If this corrects the problem, the Port Status LED will light.

  1. To check that you have not broken any network configurations rules, check the following possibilities:
    • A Fast Ethernet hub is connected to an Ethernet NIC, or a 10/100 NIC that has not been correctly configured.
    • An Ethernet hub has been connected to a Fast Ethernet hub, or hub that does not automatically sense the speed and duplex of the link.
    • There is more than one path to the same device on your network, creating an illegal network loop.
    • Your network breaks the configuration rules for Ethernet/Fast Ethernet. See section Networks explained.
