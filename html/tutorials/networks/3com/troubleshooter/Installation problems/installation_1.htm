DHCP Client was Unable to Obtain an IP Network Address from a DHCP Server:

The PC or laptop has been set up to get an IP address from a DHCP server, but was unable to get the IP address. This could be because your network does not have a DHCP server.

Caution: If you ignore this error message, the PC or laptop will continue to look for the DHCP server, and use up processing power. Your PC or laptop may start to appear slow or even come to a stop.

To avoid the problem, decide which of the following situations applies to you and then follow the appropriate instructions:

  • You do not need TCP/IP – you do not have Windows 98 computers on your network or a router, modem, or computer games that require IP.
  • You do need TCP/IP –- you have Windows 98 computers on your network or a router, modem, or computer games that require IP.

You do not need TCP/IP

There may be certain situations when you do not need TCP/IP support on your PC. For example, you are unlikely to need TCP/IP for networking if:

  • Your network only has Windows 95 PCs and laptop (that is, there are no Windows 98 PCs or laptops on your network).
  • You do not need to access the Internet or a remote site using a router or modem that requires IP.
  • You will not be using computer games that require IP.

If you do not need TCP/IP you can remove it by removing the association between the TCP/IP protocol and your NIC or PCMCIA card:

  1. Right-click (using the right mouse button) on the Network Neighborhood icon.

  2. Select Properties.

  3. Make sure that the Configuration tab is shown.

  4. Select the correct TCP/IP entry from the list, using the following guidelines:
  • If there is only one entry for TCP/IP, click on that entry in the list.

  • If there is more than one entry for TCP/IP, click on the entry that says:

    TCP/IP -> <name of your NIC or PCMCIA card>.

Caution: Make sure that you have not selected TCP/IP -> Dial-Up Adapter entry by mistake as this may be needed by other applications.

  1. Click Remove.

  2. Click OK.

  3. Restart your PC or laptop.

You need TCP/IP

Your PC or laptop may have been set up to automatically obtain an IP address from a DHCP server.

If your network does not have a DHCP server or there is a connection problem between your PC/laptop and the DHCP server, you may get a DHCP error message.

If you have a DHCP server and you wish your PC or laptop to get an IP address automatically from the DHCP server, check the network path between the PC/laptop and the DHCP server.

For example, if you have an OfficeConnect 56k LAN Modem, it contains its own DHCP server. If the computer is directly connected to the LAN modem, the problem could be with the connecting cable or the LAN modem. If you are connected to the LAN modem via a hub, you will also need to check your hub and cable connections.

If you do not have a DHCP server you will have to manually assign an IP address to your PC or laptop.

Note: for users who have both Windows 95 and Windows 98 computers on their networks. For compatibility with computers running Windows 98, the subnet mask should be 255.255.0.0 and the IP address should be 169.254.xxx.xxx (where xxx must be a number between 1 and 254), for example, 169.254.1.1. The IP address for each computer must be unique. We suggest you keep a record of the address allocated to each computer.

If you need to set up IP for a LAN Modem go to Setting Up TCP/IP. For all other devices, go to Setting up IP on a PC or Laptop.
