General Network Response is Slow for One Person Only:

If only one person on the network is experiencing problems with slow response times the most likely causes are:

  • The cable connecting the PC or laptop to the hub may be damaged.
  • The NIC or PCMCIA card is faulty.
  • The PC does not have enough processing or memory resources for the type of work the user wants to perform.
  1. Replace the cable with one known to work.

  2. If that does not solve the problem, check that the NIC or PCMCIA card is working correctly.

    • For NICs refer to Setting up a PC, Adding a PC, to check that the NIC is installed and configured correctly.
    • For PCMCIA cards refer to Setting up a PC, Adding a Laptop, to check that the PCMCIA card is installed and configured correctly.

  3. If this does not solve the problem, check that you have sufficient resources for the type of work you wish to perform.
    You may need more processor power, disk space or memory than is currently available on your PC or laptop.
    You may have a task that is running in the background, which is taking up a lot of processor power. For example:
    Your PC or laptop may have been set up to automatically look for an IP address. The PC or laptop will continually try to get an IP address, and use processor power in the process. The PC or laptop may appear slow or even come to a stop. You may see the DHCP error message described in Troubleshooting, Windows 95 and Windows 98 Error messages.

    Refer to the user guides that accompany your PC, Windows 95 or Windows 98 operating system, and applications for more information.
