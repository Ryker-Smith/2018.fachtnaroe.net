You Cannot See the PC Attached to the Printer in the Network Neighborhood:

There may be several reasons why you cannot see the PC attached to the printer in the Network Neighbourhood:

  • The PC and attached printer are not ready
  • The PC or laptop is not set up for file sharing
  • The Microsoft Browse Master has not had time to recognize the new device
  • The computers may be using different network protocols
  • There is a problem with the communication path between the devices
  1. Check that the PC with the attached printer is turned on.

  2. Check that the Printer is switched ON and on-line. If it is, go to the next page. If the printer is off-line or showing an error message, refer to the user guide that accompanies your printer to correct the problem.

  3. Make sure that the PC or laptop you are trying to access is set up for file sharing, and has the same network name as other computers on your network. To do this, carry out the following instructions:
    1. From the Start menu, choose Settings and then Control Panel.

    2. In the Control Panel window, double-click Network.

    3. In the Configuration tab, click File and Print Sharing.

    4. In the File and Print Sharing dialog box, select both of the checkboxes and click OK.

    5. Select the Identification tab.

    6. Check that there is a unique name in the Computer name field.

    7. The name entered in the Workgroup field must be the same workgroup name entered on other PCs and laptops on the network.

    8. Click OK, restart the PC or laptop and go to the next page.

  1. When a new device is added to the network, it may take the Microsoft Browse Master some time to recognise that the new device has been added. Instead of waiting for the your PC's network information to be updated, you can use the Find option to find a specific PC or all PCs on your network by carrying out the following instructions:
    1. Click on the Start Menu.

    2. Click on the Find option.

    3. Click on Computer option.

    4. In the Named field enter the name of the computer you are looking for or enter a star (*) to find all computers on the network. The results of the search are shown in the Find listbox. If the computer is not shown, go to step 6.

  1. If you have just added one of the computers to your network it is possible that the computers are using different network protocols and cannot therefore communicate with each other. For example, a PC that is using TCP/IP (usually a Windows 98 PC) will not appear in the same Network Neighborhood as a PC using NetBEUI (usually a Windows 95 PC).

    Note: You may wish to configure your Windows 95 computers to use TCP/IP, especially if they need to communicate with Windows 98 computers or with routers and modems that require IP. Setting up TCP/IP is described in Setting up IP in this OfficeConnect Network Assistant.

  2. To check the communication path between the devices that are trying to communicate, carry out the following steps:
    1. Make sure that your own and the remote PC or laptop are still connected to the hub, and that the ends of the cable are firmly pushed into the sockets on the NIC/PCMCIA card and hub.

    2. Check that the Power LED on the OfficeConnect device is lit (refer to the user guide or tutorial if necessary).
      If it is not lit, make sure that:
      • A 3Com approved and working power cable is connected to the OfficeConnect device. Refer to your supplier for details on how to purchase the correct power cable.

      • The power cable is also connected to a working main power supply socket.

        Tip: You can check that a socket is working by plugging a device you know that works into the socket, for example, a lamp.

      • If the main power supply socket has a switch, make sure that it is switched to the ON position.

      • If this does not solve the problem, the hub may be faulty. Disconnect the device from the power supply, and refer to your supplier for more details on warranty and service.

    3. Check that the Port Status LED on the hub is lit for both your own computer and the remote PC connection. If either of the Port Status LEDs are OFF, or Flashing Green for the Dual Speed Hub 8, or Yellow then carry out the instructions below:

      • Replace the cable between that PC/Laptop and the Hub with a cable you know works. If the LED then comes on, the original cable was broken.

      • Make sure that you are using the correct cable type. A common mistake is to use telephone cables. Check the table below to see which cable you should be using.

      • If the PC or laptop is connected to the highest numbered port on the hub, the MDI/MDI-X switch may be incorrectly set. Toggle the MDI/MDI-X switch to see if the Port Status LED lights.

      • Make sure that the NIC or PCMCIA card and driver software is correctly installed. If necessary, de-install the NIC/PCMCIA card and drivers, and re-install them. Refer to the installation instructions.

     

    1. Check that there are no other broken cables between the devices that are trying to see each other on the network.

    2. If there is more than one hub, check the MDI/MDI-X switch is in the correct position, and continue to the next page.
  1. Check that you have not broken any network configurations rules. The problem might be that:

    1. A Fast Ethernet hub is connected to an Ethernet NIC, or a 10/100 NIC that has not been correctly configured.

    2. An Ethernet hub has been connected to a Fast Ethernet hub, or hub that does not automatically sense the speed and duplex of the link.

    3. There is more than one path to the same device on your network, creating an illegal network loop.

    4. Your network breaks the configuration rules for Ethernet/Fast Ethernet. Refer to the section Networks Explained.
