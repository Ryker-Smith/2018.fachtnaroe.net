HP Print Server:

This section describes how to diagnose and correct problems associated with the HP JetDirect print server.

Problems are divided into four main areas:

  • Printer problems.
  • HP JetDirect hardware installation and connection problems.
  • HP JetAdmin software installation and configuration problems.
  • Network related problems.

To troubleshoot your HP JetDirect print server, you may need the following items:

  • Your printer's user guides.
  • Your printer's getting started guide.
  • The hardware and software installation manuals for your print server.
  • The diagnostics tools and utilities provided with your network software (such as the PCONSOLE or NWADMIN utility provided with Novell NetWare software, or the ping command provided with UNIX systems).

Note: Frequently asked questions and guides for installing and configuring HP JetDirect print servers can be found on HP's Electronic Support Center at: http://www.hp.com/go/support.

  1. Check that the Printer is On and Online:
    1. Make sure the printer is plugged in and turned on. If the printer will not power up, you may have a faulty power cable, power supply, or printer.

    2. Make sure that the online light is lit. If it is not, press the appropriate key to place the printer online.

    3. On printers that have a form feed light see if the form feed light is on. If it is on, it may indicate that a Form Feed was not sent with the print data, and that data is waiting for a Form Feed to print. Take the printer offline, press Form Feed, then put the printer back online.

    4. On printers that have displays, if the printer's control panel display is blank, make sure that the printer is turned on and that the HP JetDirect print server is installed correctly.

  1. If possible, print a Configuration Page. The configuration page (also called a self-test page or configuration plot) is an important troubleshooting tool. The information on the configuration page describes the status of your network and the HP JetDirect print server. The ability to print a configuration page indicates that the printer is operating correctly.

See your printer manual or the hardware installation guide for your print server for specific instructions on how to print a configuration page.

If you cannot print a configuration page:

  • The printer may be already printing a document, in which case wait until the print job is complete and then resend the configuration page.

  • There may be a problem with the printer. If the printer has a printer control panel display check for error messages.

Resolving Printer Display Error Messages

Check the following information to resolve network related error messages that appear on the printer control panel display:

40 ERROR

  1. The HP JetDirect print server detected a break in the data communications. When this error occurs, the printer goes offline. A break in communications may result from the physical network connection being disrupted or the server going down.

  2. If your printer has "auto continue" and it is set to "OFF," you must press the appropriate key (CONTINUE, for example) on the printer after the communications problem is solved to put the printer back online. Setting auto continue to ON forces the printer to reconnect without user intervention. However, this does not solve the disconnect problem.

initializing ,1,7 (INIT)

  1. This is a normal message. Wait about 3 minutes for the message to clear, or another message to appear. If another message appears, see the printer manual or the configuration page for additional information.

80 MESSAGE

  1. Turn the printer off, then on again. If the message reappears, reinstall the HP JetDirect print server. If the service message persists try the following procedure:
    • Turn the printer off.
    • Remove the HP JetDirect print server from the printer.
    • Turn the printer on again.
  1. If the message persists after you have removed the HP JetDirect print server, the problem may be with the printer. See the printer manual for more information.

  2. If there is an 80 SERVICE message, the problem is with the HP JetDirect print server and you may need to replace it. See the replacement instructions in your warranty. Depending on the type of problem that causes the "80 SERVICE" message, the print server may send a diagnostics page to the printer when the error occurs. If you get multiple diagnostics pages for a single failure, forward these pages to your HP authorized dealer for further analysis. If you need to replace the print server, include the diagnostics pages with the old print server.

See your printer documentation for a complete list of control panel messages and corrective actions.

Checking HP JetAdmin Software Installation and Operation

Refer to the printer configuration page where necessary (also called a self-test page or configuration plot). See the HP JetAdmin README file for more information.

  1. If you are having problems installing the HP JetAdmin software on your computer, refer to the HP JetAdmin user guide:
    1. Check that your computer meets the memory requirements for the software.

    2. Check that your network operating system and hardware platform are supported by the HP JetAdmin software.
    3. If necessary, reinstall the JetAdmin software.

  1. If you are having problems running the HP JetAdmin software:
    1. Make sure you are logged in with supervisor access.

    2. Try running the JetAdmin software from a different computer to check that the problem is not with your computer.

    3. Make sure your system software and computer requester software are up to date.

    4. Reinstall the JetAdmin software.

    5. Check that the media the HP JetAdmin software came on is not corrupt or damaged.

    6. Make sure that any changes made recently on the network were performed correctly.

    7. If hardware has been recently moved or added to the network, check that it was moved, deleted, or installed correctly. Refer to the appropriate chapter in your user guides for information about how to check the connection.

    8. If any configuration files have been modified recently, make sure the changes were correct and that the HP JetDirect print server configuration reflects those changes. Refer to the appropriate chapter in your user guides.

  1. If the HP JetDirect print server is not communicating correctly with the HP JetAdmin software:
    1. Check that the correct print server name appears next to NODENAME on the configuration page. If not, refer to your user guides for installation information.

    2. Check all of the settings on the configuration page.

    3. Make sure that any changes made recently on the network were performed correctly.

    4. If hardware has been recently moved or added to the network, check that it was moved, deleted, or installed correctly Refer to the appropriate chapter in your user guides for information about how to check the connection.

    5. If any configuration files have been modified recently, make sure the changes were correct and that the HP JetDirect print server configuration reflects those changes. Refer to the appropriate chapter in your user guides.

    6. If this fails to solve the problem refer to Printer Communication problems with the Network.

  1. If the setting you want in the HP JetAdmin software is missing or grayed out:
    1. The printer or HP JetDirect print server does not support this functionality or

    2. You may not be logged in with the appropriate privileges on the network.

  1. If you cannot configure the printer after selecting the printer address in HP JetAdmin software see the online help included with the HP JetAdmin software.

Printer Communication Problems with the Network

If you are having problems printing to the printer over the network:

  1. Check the physical connection between the computer or file server and the HP JetDirect print server. Check network cabling, connections, and router configurations.

  2. Make sure that the printer is attached to the network using the appropriate HP JetDirect print server port and cable. Check each cable connection to make sure it is secure and in the right place. If the problem continues, try a different cable or ports on the hub or transceiver.

  3. Check that your network is terminated correctly. If you are using a ThinLAN (thin Ethernet coaxial cable), your network must form a line, not a loop. If your printer is the end node, the BNC connector must have a 50-ohm terminator attached to one end.

  4. If you have added any software application to the network, make sure they are installed correctly and use the correct drivers. Make sure that the software is compatible with other software on the network.

  5. If other users can print, the problem is probably computer specific. Check the computer network drivers, printer drivers, and redirection (capture in Novell NetWare).

  6. If other users are able to print, are they using the same network operating system? Refer the online help provided with HP JetAdmin software.

  7. Check the STATUS line for your protocol on the configuration page to make sure that it is enabled.

  8. If there is an error message in the protocol's section on the configuration page refer to the configuration page section of your user guide for a list of error messages and actions.

  9. If you are using Novell NetWare, does the printer (node address) appear in the HP JetAdmin software?
    1. Check network and HP JetDirect settings on the configuration page.

    2. Confirm the network settings for the printer using the printer control panel (for printers with control panels).

    3. See the troubleshooting section in the online help of the HP JetAdmin solution for your network operating system.

  1. If you are using Apple EtherTalk or LocalTalk, does the printer appear in the Chooser?
    1. Check network and HP JetDirect settings on the configuration page.

    2. Confirm the network settings for the printer using the printer control panel (for printers with control panels).

    3. See the troubleshooting section in the online help for the HP LaserJet Utility.

  1. If you are using IBM LAN Server/Warp Server, does the printer (node address) appear in the HP SETUP or Port Settings utility?
    1. Check network and HP JetDirect settings on the configuration page.

    2. Confirm the network settings for the printer using the printer control panel (for printers with control panels).

    3. See the troubleshooting section in the online help included with the HP JetAdmin software for your network operating system.

    4. If the printer does not show up, check that it is not already configured.

  1. If you are using Microsoft Windows NT, does the printer appear in the HP JetAdmin software?
    1. Check the network and HP JetDirect settings on the configuration page.

    2. Confirm the network settings for the printer using the printer control panel (for printers with control panels).

    3. See the troubleshooting section in the online help included with the HP JetAdmin software for your network operating system.

  1. If you are using Microsoft Windows NT 4.0, does the printer appear in the Add a Hewlett-Packard Network Peripheral Port dialog box?
    1. Check the network and HP JetDirect settings on the configuration page.

    2. Confirm the network settings for the printer using the printer control panel (for printers with control panels).

    3. See the troubleshooting section in the online help included with the HP JetAdmin software for your network operating system.
