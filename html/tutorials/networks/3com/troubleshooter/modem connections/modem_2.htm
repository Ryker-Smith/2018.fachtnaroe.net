No Dial Tone Available for an Attached Analog Telephone:

  1. If there is a call in progress (Multilink PPP for the ISDN LAN Modem; single call for the 56k LAN Modem) disconnect the call. Disconnecting a call provides an analog channel for the attached analog device (in this case the attached telephone).

  2. If you still cannot get a dial tone, make sure that the analog phone and all cables are connected correctly.

  3. If you have the OfficeConnect 56k LAN Modem refer to your User Guide for technical support information. If you have an ISDN LAN Modem there are two configuration related reasons why you may not hear a dial tone:

    • Call routing is configured so that calls are blocked to an analog port(s)

    • The service provider connection is configured to always use both B channels thus preventing an available channel for a voice call when a data call is connected.

    • The next two steps describe how to check each of these settings.
  1. It is possible to block calls on an analog port so that calls cannot be made on that port. Check that the phone is not connected to a blocked port. Access the Call Routing window in the Web Wizard. Make sure that the correct Phone Port box for the attached analog device is selected.

  2. If this does not solve the problem, check to see if the Service Provider in the Web Wizard is set to "Always use 2 B channels". If it is, change the bandwidth allocation to either "Add second B channel as needed" or "Always use 1 B channel".

  3. If you still cannot get a dial tone, there may be a problem with the connected analog phone. If possible test the analog phone by connecting it to an analog phone wall socket and make a call, or use an analog phone that you know works.
