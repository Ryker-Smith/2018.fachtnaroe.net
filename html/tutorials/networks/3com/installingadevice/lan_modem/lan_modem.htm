1 Overview

1.1 This subsection describes how to set up an OfficeConnect LAN Modem and establish a connection to the Internet.

The OfficeConnect LAN Modem is available in three different variations:

  • The 56K LAN Modem that uses an analog connection to access the WAN
  • The Dual 56K LAN Modem that uses two analog lines to access the WAN
  • The ISDN LAN Modem that uses an ISDN line to access the WAN

More information about each of these LAN Modems, including complete documentation, can be obtained by visiting the OfficeConnect product web site at:

http://www.3com.com/support/docs/lanmodem

The LAN Modem connects directly to your computer's 10BASE-T Ethernet NIC or your laptop's 10BASE-T PCMCIA card, which must be installed on each computer you want to connect to the LAN Modem. Please contact your supplier or nearest 3Com Sales Office for availability in your region. Information on installing a NIC into your PC or laptop can be found under Setting Up a PC.

1.2 To set-up the LAN Modem for Internet access, you need:

  • The OfficeConnect LAN Modem, the power adapter, telephone cable and the 10BASE-T Ethernet cable that accompanied the unit.
  • An Internet Service Provider (ISP) account (unless you are only connecting to a private network).
  • A JavaScript-enabled, frames-capable Web browser, such as Microsoft Internet Explorer (3.0 or later) or Netscape Navigator (3.0 or later).
  • TCP/IP software installed and setup on your computer. TCP/IP is included as part of the Windows 95/98 Operating System.
  • An available telephone outlet appropriate for your LAN Modem type (analog or ISDN). For more information on cabling, refer to Connectors and Cabling.

Setting up the LAN Modem requires three steps:

1. Physical installation.
2. Setting up TCP/IP.
3. Configuring the LAN Modem's WebWizard for Internet access.

2 Physical installation

2.1 To physically install the LAN Modem you should:

  • Install the telephone cable.
  • Connect the LAN Modem to a computer.
  • Connect any analog equipment to the LAN Modem.
  • Connect the power cable.
  • If necessary, connect the LAN Modem to an Ethernet hub on your network.

Note: Before you begin, turn off the computer(s) you will connect to your LAN Modem.

2.2 Installing the Telephone Cable

The type of telephone cable you install will vary depending on the type of LAN Modem you are using.

For the 56K LAN Modem, connect one end of the RJ-11 analog cable to the port labeled Line on the back panel. Then connect the opposite end of the RJ-11 cable into an available POTS (plain old telephone system) analog wall jack. For the Dual 56K LAN Modem you can connect an additional RJ-11 analog cable to the Line 2 port on the back panel to obtain dual analog access.

The U version of the ISDN LAN Modem is for North America and other countries that do not require the use of an external NT-1. Connect the RJ-45 (8-pin) connector end of the RJ-45 /RJ-11 ISDN cable to the port labeled ISDN on the back panel. Then connect the RJ-11 (6-pin) connector end of the RJ-45/RJ-11 ISDN cable to the
RJ-11 or RJ-45 ISDN wall jack.

The S/T version of the ISDN LAN Modem is for Europe and other countries that require the use of an external NT-1. Connect the RJ-45 (8-pin) connector end of the RJ-45/RJ-45 ISDN cable to the port labeled ISDN on the back panel. Then connect the opposite end of the RJ-45/RJ-45 ISDN cable into an ISDN wall box (NT-1).

2.3 Connecting the LAN Modem to a Computer

The LAN Modem must be initially configured when attached only to one computer. To connect the LAN modem to your computer, insert one end of the 10BASE-T Ethernet cable into any one of the four LAN ports on the back panel of the LAN Modem.

Then insert the opposite end of the cable into your computer's 10BASE-T Ethernet port.

2.4 Connecting analog equipment to the LAN Modem

You can connect an analog touch-tone telephone, answering machine, fax machine, or external analog modem to the LAN Modem's analog ports. The ISDN LAN Modem and Dual 56K LAN Modem provide two analog ports for connecting analog devices. The 56K LAN Modem provides one. You will need the RJ-11 to RJ-11 analog cable that accompanied the analog device for each analog phone port connection.

If the modem has more than one spare analog port you can place more than one call at a time. For example, one user may connect to a remote office LAN to check e-mail while another connects to a locally dialed ISP to browse the Web. The 56K LAN Modem provides one analog port, and does not support simultaneous voice and data calls.

To install an analog device, insert one end of an RJ-11 analog cable into the port labelled Phone on the back of the LAN Modem. Then insert the other end of the RJ-11 cable into the appropriate RJ-11 port on the analog device.

2.5 Connecting the Power Cable

You must use the power adapter that was supplied in the LAN Modem package. To install the power cable, connect the LAN Modem power module cable to the 10-18V DC power connector on the back panel of the LAN Modem. Plug the other end of the power module into a surge-protected standard wall outlet. This causes the indicator LEDs on the LAN Modem's front panel to flash momentarily, as the unit undergoes a power-up self-test diagnostic. If the LEDs do not flash, go to the Troubleshooting section.

The next step is to set up the LAN Modem for WAN access.

2.6 Connecting the LAN Modem to an Ethernet Hub

You may want to connect the LAN Modem to an external Ethernet hub, allowing WAN access for up to 25 users. To do this, you need the following:

  • An external hub (for example, an OfficeConnect hub).
  • A 10BASE-T Ethernet cable, which may have been provided with the additional hub. If the hub does not have an Uplink/Normal (MDI/MDI-X) switch, you must use a crossover cable.

To install the external hub, insert one end of the 10BASE-T Ethernet cable into one of the four LAN ports on the back of the LAN Modem. Then insert the opposite end of the cable into the Uplink port of your hub.

2.7 The next step is to set up the LAN Modem for WAN access.

3 Setting up TCP/IP

3.1 The LAN Modem is optimized for use with dynamic IP addressing, which means that the LAN Modem will function as the DHCP server. In this situation, TCP/IP must be installed and set to its default values. To do so:

1. Click Start, Settings, Control Panel.
2. Choose the Network control panel.

If TCP/IP is shown in your network components list, it is already installed. In this case, click here for instructions on setting TCP/IP to its default values. If TCP/IP is not shown, the following steps explain how it can be installed.

From the Network control panel click Add.

The Select Network Component Type dialog box opens.

Choose Protocol and then click Add.

The Select Network Protocol dialog box opens.

From the Manufacturers list box, select Microsoft, and then from the Network Protocols list box, choose TCP/IP.

Click OK to return to the Network control panel.

3.2 Setting TCP/IP to the Default Values

You can verify that your TCP/IP values are set to their defaults from within the Network control panel.

If it is not currently open, click Start, Settings, and Control Panel and choose Network to open the Network control panel.

1. Highlight TCP/IP and click Properties.

2. Click the IP Address tab. Obtain IP address automatically should be enabled.

3. Click the WINS Configuration tab. Use DHCP for WINS Resolution should be enabled.

4. Click the DNS Configuration tab. Disable DNS should be selected.

5. Click OK to close the TCP/IP Properties window.

6. Click OK to close the Network control panel and save any changes if prompted.

7. You must now restart your PC or Laptop.

5 Disconnecting the call

5.1 The connection established as a result of the ISP Wizard will automatically disconnect after a designated period of time. This type of Automatic Call is placed every time you launch a WAN application, such as a Web browser to access the Internet, or an e-mail application to read e-mail.

The length of time for this call varies depending on the LAN Modem version you are using and the time you have designated. You can manually disconnect a call at any time via the Manual Calling screen, which is accessed from the LAN Modem's main page.

6 Setting up dial-in support on your LAN

6.1 If you have an OfficeConnect ISDN, 56K, or Dual 56K LAN Modem, you can place outgoing calls from your local LAN to an existing network (such as the Internet or a private, corporate network), or receive calls into your LAN from up to 10 specified remote users.

A LAN Modem can only accept dial-in calls that match its call type. The 56K and Dual 56K LAN Modem can only accept dial-in calls from another analog device, such as a regular analog modem, a 56K LAN Modem or second Dual 56K LAN Modem, or other analog router. It cannot accept dial-in calls from an ISDN terminal adapter or ISDN LAN Modem. Similarly, the ISDN LAN Modem can only accept dial-in calls from another ISDN device.

The term 'server-side' is used to describe a LAN Modem that receives dial-in calls. The term 'client-side' is used to describe a LAN Modem that makes dial-out calls. In some cases, the same LAN Modem will be configured as both a server-side LAN Modem and a client-side device.

For more information on setting up an OfficeConnect LAN Modem, refer to Installing an OfficeConnect LAN Modem.

6.2 Setting up a LAN Modem for dial-in support is a two-part process:

  • Configuring the server-side LAN Modem to receive incoming, dial-in calls.
  • Configuring a client device (such as an analog modem or second LAN Modem) for dial-out access to the server-side LAN Modem.

In order to access LAN resources remotely, any workstation, server, or device attached to the LAN Modem that will be accessed by a dial-in user must be running appropriate software, such as Web or FTP server software.

The LAN Modem provides three dial-in profile types:

  • Single User - This profile type applies to a single user dialing in to the LAN Modem via a single analog modem. It is used to set up a connection between a server-side LAN Modem and a client device that does not require LAN-to-LAN connectivity.
  • LAN Modem Site-to-Site - This profile type applies to any LAN Modem that will place outgoing calls and receive incoming calls from another LAN Modem or LAN Modems. It is used to set up a network between two or more sites, using a LAN Modem at each site to create true LAN-to-LAN connectivity. This enables any device on any LAN to communicate directly with any other device on any other LAN.
  • Advanced - This profile type applies to a network scenario in which a router other than a LAN Modem will be dialing into the LAN Modem. It assumes that both the LAN Modem and the other router have been manually configured for static IP addressing, and each employs a unique range of addresses. For more information on dial-in user profiles, refer to the Release Notes that accompany your product.

6.3 The first step in configuring the server-side LAN Modem for dial-in support involves setting up the LAN Modem to receive incoming, dial-in calls. This process involves three steps:

  • Changing the Remote Administration Username and Password
  • Configuring Dial-in Global Parameters
  • Creating a Dial-in User Profile Using the Dial-In Wizard

6.4 Changing the Remote Administration Username and Password

The Remote Administration account allows for limited dial-in access to the LAN Modem for remote configuration purposes only. A Remote Administration account is already set up in the LAN Modem, with the following defaults:

Username: Admin
Password: 1234

Before you can configure the LAN Modem for full dial-in access, you must change these defaults:

1. From the LAN Modem Main Configuration Page, click Dial-in, then click Dial-in User. The Dial-in User Selection window opens.

2. Select Admin, and then click Select. In the Username field, delete the default name, and enter a unique Username. In the Password field, delete the default password and enter a new Password. When you configure a client device for dial-out access to this LAN Modem, you will need this Username and Password.

3. Click Submit. You can now configure dial-in global parameters, discussed in the next section.

6.5 Configuring Dial-in Global Parameters

Dial-in Global Parameters allows you to allocate modem line resources for dial-in calls and to set dial-in authentication preferences. To do so, follow these steps:

1. From the LAN Modem Main Configuration page, click Dial-In. Click Dial-in Global. The Dial-In Global Parameters window opens.

2. Select 1 in the Channels allocated for Dial-in field to allocate only one telephone line for dial-in calls (the default). Select 2 to allow both telephone lines to be used for dial-in calls, either by a single user making a Multilink PPP call, or by two users dialing in simultaneously. (Note that this option is not available for the 56K LAN Modem, which uses only one telephone line.)

3. Check the Allow Multilink field box to allow both telephone lines to be used by a single caller for a Multilink call, or leave the box unchecked to allow only one line per dial-in call. (Note that this option is not available for the 56K LAN Modem, which uses only one telephone line.)

If a telephone number prefix, such as a 9, is required to reach an outside line, enter this number in the Callback Prefix field. Otherwise, leave this field blank.

4. In the LAN-to-LAN Site Number drop-down list box, choose one of the following options to define this LAN Modem and its corresponding LAN within the overall network:

  • Choose None if you are configuring this LAN Modem to only receive dial-in calls from a Single User Dial-in client device Choosing None will leave the LAN Modem's IP address unchanged. (Advanced users may want to choose None if the LAN Modem has already been manually configured for static IP addressing.)
  • Choose Site A if you are setting up a site-to-site network with two or more LAN Modems, and this is the first LAN Modem that you are configuring (in other words, this LAN Modem will place and/or receive calls to another LAN Modem).
    Choosing Site A will leave the LAN Modem's IP address set to the default of 192.168.1.1, unless the IP address has been changed. If changed from its default, the IP address will revert back to 192.168.1.1.
  • Choose Site B if you are setting up a site-to-site network with two or more LAN Modems and you have already designated another LAN Modem as Site A. Choosing Site B will cause this LAN Modem to change its IP address to 192.168.2.1.
  • Choose Site C if you are setting up a site-to-site network with two or more LAN Modems and you have already designated another LAN Modem as Site A and/or Site B. Choosing Site C will cause this LAN Modem to change its IP address to 192.168.3.1.

In the PPP Authentication field, choose the type of PPP authentication you want the LAN Modem to use when negotiating all dial-in calls. Click Submit to save your changes.

If you choose Site B or Site C as the LAN to LAN Site Number, the LAN Modem will re-initialize itself as it changes its IP address. If you chose Site B or C, re-start your workstation to allow it to acquire a new IP address from the LAN Modem's changed subnet.

This completes the Dial-in Global Parameters portion of the server-side configuration. You can now create a dial-in user profile using the Dial-In Wizard in the next section. A separate Dial-in User Profile must be configured for each device that will dial in. In the case of a second LAN Modem or other router, one Dial-in User Profile applies to the LAN Modem and its entire attached LAN.

6.6 Creating a Dial-In User Profile Using the Dial-In Wizard

To use the Dial-in Wizard to create a single user dial-in profile, perform the following steps:

From the LAN Modem Main Configuration Page, click Dial-In, then click Dial-In Wizard. The Dial-In Wizard Profile Type Selection window appears.

Select Single User from the drop-down list, and then click Next. The Dial-In Wizard Password window appears.

Enter the unique Username and Password the dial-in user must enter when calling in to this LAN Modem. (This information will be entered in the Username and Password fields when you configure your client device.)

Click Next. The Dial-In Wizard Internet Service Provider Selection window appears. Select an Internet Service Provider for this dial-in user to use when accessing the Internet from the remote location. Click Finish.

This completes the server-side portion of the configuration process. To configure the client-side device for dial-out access to this LAN Modem, refer to your LAN Modem manual for the section on setting up a client device for dial-out access, or visit the LAN Modem support site at:

http://www.3com.com/support/docs/lanmodem

If you wish to use the LAN Modem's Callback feature, you must configure callback parameters for this LAN Modem from the Dial-in User Parameters page, discussed in section 7.2.

6.7 Dial-In Wizard for LAN Modem Site-to-Site Dial-In

To create a Dial-in User Profile to allow access by a second site LAN Modem, follow these steps:

1. From the LAN Modem Main Configuration Page, click Dial-In. Click Dial-In Wizard. The Dial-In Wizard Profile Type Selection window appears.

2. Select LAN Modem Site-to-Site from the drop-down list, and then click Next. The Dial-In Wizard Password window appears.

3. Enter a unique Username and Password for the second site LAN Modem. (This information will be entered in the Username and Password fields when you configure a Private Network Service Provider profile in the second site LAN Modem.) Click Next. The Dial-In Wizard Site Assignment window appears.

4. Using the drop-down list box assign a Site identity to the remote, second site LAN Modem. For example, if you designated this LAN Modem as Site A, designate the second site LAN Modem as Site B or Site C. Click Next. The Dial-In Wizard Internet Service Provider Selection window appears.

5. Select an Internet Service Provider for this dial-in user to use when accessing the Internet from the remote location. Click Finish.

This completes the server-side portion of the configuration for LAN Modem Site-to-Site Dial-in. To finish the LAN Modem Site-to-Site configuration, you must also configure this LAN Modem as a client-side device. You must also configure the second site LAN Modem as both a server-side LAN Modem and a client-side device, following the same procedure that you used to configure this LAN Modem.

6.8 Dial-in Wizard for Advanced Dial-In

To create a Dial-In User Profile for Advanced Dial-In, follow these steps:

1. From the LAN Modem Main Configuration Page, click Dial-In. Click Dial-in Wizard. The Dial-In Wizard Profile Type Selection window appears.

2. Select Advanced from the drop-down list, and then click Next. The Password page appears. Enter a unique Username and Password to identify and verify this analog router when it calls in to the LAN Modem. This information must be entered in the analog router when it is configured for dial-out calling to this LAN Modem.

3. Click Next. The Dial-in User Parameters page appears.

7 Configuring dial-in parameters

7.1 Once you have created a Dial-in User profile using the Dial-in Wizard, you can further define the dial-in user parameters using the Dial-in User Parameters page. You can also use this page to configure an Advanced Dial-in User profile, or to create any Dial-in User profile without the Dial-in Wizard. You can create a total of 10 dial-in user profiles.

Click Dial-in Users. The Dial-in User Selection screen opens. Choose either New Dial-in User to create a new profile, or choose an existing Dial-in User from the drop-down list, and click Select.

The Dial-in User Parameters screen opens. In the Name field, enter a unique name to designate this dial-in user. In the Password field, enter a password to verify access when this user dials into the LAN Modem. In the Type of Use field, select the scenario for which your dial-in situation applies:

  • Choose Single-User Dial-in if this user will be accessing the LAN Modem from a regular analog modem or second analog LAN Modem on which attached workstations can share a single IP address using NAT.
  • Choose LAN Modem Site-to-Site if this user will be dialing in from another LAN Modem configured as both a server-side and client-side device (for "peer-to-peer" networking).
  • Choose Advanced if this user will be accessing the LAN Modem from a router other than a LAN Modem. This scenario requires that all IP addresses be statically configured, and should be configured by your system administrator.

In the Idle Timer field, enter the length of inactivity before the LAN Modem drops the incoming call. By default this value is 300 seconds (5 minutes). In the ISP Selection field, select the service providers that this dial-in user can access. In the Data Compression field, choose to have data compression active or inactive. By default compression is disabled. (In order for compression to occur, the other end of the connection must also have compression active.)

7.2 Callback Parameters

Callback occurs when, upon a successful connection of dial-in a user, the LAN Modem drops the incoming call and places a return call to the dial-in user. In order for callback to work properly, the client-side device must also have callback active. To specify your Callback parameters, follow these steps:

1. In the Enable Callback field, check to activate Callback for this dial-in user. By default, Callback is disabled. In the Callback Telephone Number field, enter the number the LAN Modem will call back when this dial-in user accesses the LAN Modem. In the Callback Alternate Number field, enter a second telephone to be used for either Multilink PPP or as a second number to try if the call cannot be established with the first telephone number.

2. In the Callback Authentication field, check the box to require callback authentication if requested by the client. In the Callback Username field, enter the username used by the server. In the Callback Password field, enter the password used by the server.

3. In the Callback Delay field, enter the amount of time in seconds to wait until the dial-in user is called back. By default this is set to 10.

7.3 WAN Parameters

If the local LAN Modem WAN link has a static IP address, enter the WAN static IP address in the LAN Modem WAN IP field. Otherwise leave this field as set by the Dial-in Wizard. If you are using the Advanced dial-in scenario, enter the IP address of the LAN Modem which is receiving the dial-in call.

If the remote site of the WAN link has a static IP address, enter the WAN static IP address in the Remote LAN IP field. Otherwise, leave this field as set by the Dial-in Wizard. For the Advanced dial-in scenario, enter the IP address of the router dialing into the LAN Modem.

If the remote site of the WAN link has a static IP address, enter the subnet mask of the remote LAN in the Remote LAN Subnet Mask field. Otherwise, leave this field as set by the Dial-in Wizard. If you are using the Advanced dial-in scenario, enter the subnet mask of the remote LAN.

Enter your Primary and Secondary (optional) DNS address as instructed by your system administrator, or as set by the Dial-in Wizard. For the Advanced dial-in scenario, enter the IP address of the LAN Modem receiving the dial-in call.

7.4 Placing a Call from a Client-side LAN Modem

Once you have created a service provider profile on your client LAN Modem, you can place a call in one of three ways:

  • Use the client LAN Modem's Manual Call Control page to place a call manually.
  • Use the LAN Modem Desktop Manager on a client workstation to place a call manually.
  • Enter an IP address or domain name of a server attached to the remote LAN Modem, and place the call automatically. Note that this option assumes that you have configured a Private Network as your service provider, and not an ISP.

Once connected, you can use all LAN resources available on the server end of the connection. For further information, visit the OfficeConnect LAN Modem's Internet Homepage at

http://www.3com.com/support/docs/lanmodem
