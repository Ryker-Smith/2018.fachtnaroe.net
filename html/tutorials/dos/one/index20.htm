Capturing Printer Output to Graphics
 
  
 
Three files are discussed here. The first, FileCap, is a Windows 95 program for capturing serial data. The second, Prn2Pbm, is another Windows 95 program which converts captured PRN files to the PBM graphic format. Lastly, DosCap is a collection of DOS programs which perform both the above functions (but only under DOS 5 and 6). 
Why did I write these programs? Simple. I use several pieces of test equipment which can do a "Print Screen" out to a serial printer. The problem is, I wanted to embed pictures of these screens in Word documents and web pages. To top it off, I don't have a serial printer. So I had to capture the test equipment output onto my PC, then convert that captured file into a recognizable graphic. I wrote a separate program for each step. Here's a sample picture of an R2660 spectrum analyzer, a SunSet T1 test set, and the Windows 95 Printer Test Page.
 
FileCap,unlike most other programs, will reliably capture streaming binary data intact. Use it to capture serial printer data using a nul modem (For example the output of an R2660, SunSet T1, or other dedicated test equipment) or use it to capture EBTS data. You'll need to have previously installed a VB5 program or the VB5 runtime files. I got tired of install programs failing, so none is included.


Provided you have a captured IBM-Graphics printer file, this program will convert it to the rare (but easy to program for) PBM file format. You'll need to have previously installed a VB5 program or the VB5 runtime files. I got tired of install programs failing, so none is included. You can use LView,  PaintShop, or a similar graphics utility to convert the PBM files to whatever you want. 


DosCap contains DOS versions of a serial capture program and a PRN2PBM conversion program. Both the executable and the QBasic source code are included -- which you'll need. The capture program has hard-coded settings for the serial port, so you're better off running the QBasic code interpreted and tweaking it as needed to get your serial ports set up. Frankly, neither of these programs was ever entirely stable, but they both did work. Both cause spectacular errors if you try to run them under Windows 95. I have no idea why...
