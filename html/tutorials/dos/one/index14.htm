FFIND
 
 
 
Here's the story: FFIND was written to solve a single problem. I couldn't find an elegant way to search one of those multi-line description BBS file lists for files that interested me. Sure, most of them are broken up into different "areas" or "directories", but even one area can get pretty big! If you try searching them with the MSDOS "FIND" command, you snag all the lines which contain the item you're interested in, BUT YOU MISS ALL THE FILE NAMES AND THE ASSOCIATED DESCRIPTIONS! 

FFIND solves that. You use it just like FIND - but with different options. FFIND searches entire BLOCKS of data instead of single lines like FIND.
