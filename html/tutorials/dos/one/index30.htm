Office File Opening Problems

Problems like this can affect all office file types, but people only seem to notice it in Excel or Word.


Have you ever seen a warning like one of these?

This is caused by Excel (or Word) not correctly recognizing long file names or not correctly handling command line arguments and DDE commands. I figured out an involved workaround, but Alexandre Fiset from Nortel pointed out a simpler solution: Launch the offending Office97 application with the /regserver command line switch. For example:
"C:\Program Files\Microsoft Office\Office\Excel.exe" /regserver
This works for Excel, Word, and Access. Rather than type in the command line, I use a shortcut, but the end result is the same -- Problem fixed. Thanks Alexandre!
