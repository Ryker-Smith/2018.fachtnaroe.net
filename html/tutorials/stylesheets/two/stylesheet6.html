-Conclusion 		LESSON FOUR: Positioning and Visibility

We mentioned before about a method that would allow you to effectively overlap items on your page. This final section is devoted to CSS properties that give you exact control over the positioning, visibility, z-index, and several other properties for elements on the page.

POSITIONING

Positioning is declared as either absolute or relative. The difference is that absolute allows you to set the exact position of the item, while relative takes into account where the elements appears in the normal layout of the page. A few examples should make this more clear.

h1 { position: absolute; left: 50 px; top: 10 px }

This line of code would position the text contained in the <h1> tag 50 pixels from the left and 10 pixels from the top of the window, no matter what. So even if this code was placed at the end of a page, the element would still appear in that position.

h4 { position: relative; left: 100 px; top: -10 px }

With relative positioning the placement of the element within the normal flow of the page is taken into account, and the position is changed accordingly. So the text in the <h4> tag would be placed 100 pixels to the left and -10 pixels from the top in its normal position within the page. Note that top and left are the only two directions specified.

Using the position properties is the most effective way to overlap text. The following lines of code is an example of how to get two headings to overlap.

h1 { position: relative; left: 100 px; top: 0 px }
h2 { position: relative; left: 75 px; top: -35 px }

Z-INDEX

This property is used to decide what element will be displayed on top of another when elements are overlapped. Simply use a higher whole number value for the element you wish to be placed in front of the other.

h1 { position: relative; left: 100 px; top: 0 px; z-index: 2 }
h2 { position: relative; left: 75 px; top: -10 px; z-index: 1 }
VISIBILITY

This property is especially useful when used in dynamic HTML, since is gives you the power to display or hide elements on your page when certain conditions are met or other actions are performed by the user.

h4 { visibility: hidden }

Setting the property to visible will obviously show instead of hide the element.
