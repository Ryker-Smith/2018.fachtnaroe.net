 LESSON TWO: Controlling Fonts

Font attributes are probably the most used of the CSS properties. These properties can be applied to everything from the anchor tag (a { font-size: 10 px }) to the various headings (h4 { text-decoration: none }). You use these anytime you want to control the properties of any tag that handles text.

This is the way that all stylesheet properties are used. First you specify which tag you wish to apply the properties to. Then you simply list all of the specific properties that you want to set for the tag. When this tag is used in the body of the HTML page, it will have all of the attributes you set for it already applied. Later in this lesson we will see how to specify multiple classes of the same tag.

The most important font attributes are as follows:

font-family
font-size
font-weight
font-style
font-variant
text-transform
text-decoration

FONT FAMILY

The most basic of the font attributes is font-family. This, as the brighter among you may have guessed, is used to define the typeface that will be used. When using this it is best to include several fonts to make sure your viewer has at least one of the specified fonts.

p { font-family: arial, verdana, sans-serif }

The browser will look at the fonts sequentially, using the first recognized font. It is good practice to include as the last choice a generic font face that serves as a default. These default names include:

serif (most likely Times)
sans-serif (probably Arial)
monospace (probably Courier)
cursive (please just don’t use this)
fantasy (again, why?)

A few tips. If your font name consists of two or more words (ex: Gill Sans) enclose it in quotes. Also, an odd quirk of stylesheets is that when you define several properties for the same tag (ex: h2 { color:green; margin: 10 px; font-family: verdana} ) make sure that the font-family attribute is the last one. It’s just one of the stupid quirks of CSS.

FONT SIZE

When you are specifying font size you have several choices. The most common methods of declaring font sizes are using points or pixels, although it is possible to use inches, millimeters or even percentage values.

When declaring the font size use pt to declare points and px for pixels.

P { font-size: 16 pt }

As you can see, no longer are you forced to use the familiar HTML sizes of 1 through 7 to declare font size. You can use 7 pt, 56 px, or even 12 in. This allows you to add a unique touch to your pages since you can use a font size that seems unique when the viewer is used to seeing the common 2’s and 3’s of HTML font size.

Hopefully you are starting to get the hang of how CSS syntax looks, so we’ll cover the next topics a little more quickly.

FONT STYLE

This handles the italic attribute of the font. You can also specify the font as normal.

h3 { font-style: italic }
FONT WEIGHT

This handles the bold attribute.

a { font-weight: bold }

You can also use a numeric value (from 100 - 900) that will handle the various levels of boldness, if the font supports it.

TEXT TRANSFORM

Converts text to uppercase.
b { text-transform: uppercase }
Converts text to lowercase.
h1 { text-transform: lowercase }
Capitalizes first letter of each word.
a { text-transform: capitalize }
Makes sure that all text is unchanged.
p { text-transform: none }
TEXT DECORATION

These are most commonly associated with the anchor tag.
a { text-decoration: underline }
a { text-decoration: overline }
a { text-decoration: line-through }
a { text-decoration: none }
Writing text rollovers with CSS is a breeze. Take a look at the following code:
a:link { text-decoration: underline }
a:hover { text-decoration: none }
a:visited {text-decoration: underline }
CLASSES

The above lines of code can now apply to every single link on your page. But what if you wanted some links to be of a different style? Well, this is where classes come into play. To define tags with various classes, simply write:

a:link.secondtype { text-decoration: underline }
or with other tags:
b.nim { color: green }
To use the different class, specify it when you write the tag in the body of your code:

<a class=”secondtype” href=”www.wildeware.com”>WildeWare Studios </a>

<b class=”nim”> www.wildeware.com </b>

Leaving out the class will use simply use the parent properties of the tag.
